# AI编程常用名词概念速查手册

> **适用人群**：使用AI工具（Trae/Cursor/Copilot等）辅助编程的产品经理和开发者  
> **更新频率**：随技术演进持续补充  
> **最后更新**：2024-04-26

---

## 一、AI工具与平台类

| 名词 | 全称/含义 | 通俗解释 | 使用场景 |
|------|----------|---------|---------|
| **Copilot** | 副驾驶/编程助手 | AI帮你写代码的搭档 | 写代码时自动补全 |
| **Prompt** | 提示词 | 你给AI的指令 | "帮我写一个登录页面" |
| **Context** | 上下文 | AI理解的"背景信息" | 打开的文件、代码库内容 |
| **Model** | 模型 | AI的"大脑" | GPT-4、Claude、通义千问 |
| **Temperature** | 温度参数 | 控制AI输出的创造性 | 0=保守精确，1=天马行空 |
| **Max Tokens** | 最大Token数 | AI单次回复的最大长度 | 限制输出篇幅 |
| **API** | Application Programming Interface | 应用程序接口，让不同软件对话 | 调用AI服务的方式 |
| **SDK** | Software Development Kit | 软件开发工具包 | 集成第三方功能的工具集 |
| **Plugin/Extension** | 插件/扩展 | 给编辑器加功能 | 在VS Code中安装AI插件 |

## 二、代码与开发基础类

| 名词 | 含义 | 通俗解释 | 示例 |
|------|------|---------|------|
| **Repository（Repo）** | 代码仓库 | 存放项目所有代码的地方 | GitHub上的项目 |
| **Commit** | 提交 | 保存一次代码改动 | "fix: 修复登录bug" |
| **Branch** | 分支 | 代码的平行版本 | main（主线）、dev（开发） |
| **Merge** | 合并 | 把分支的代码合到一起 | 把开发分支合到主线 |
| **Pull Request（PR）** | 拉取请求 | 请求别人审核你的代码改动 | 开发完成后提交PR |
| **Bug** | 缺陷/错误 | 代码中的问题 | 点击按钮没反应 |
| **Debug** | 调试 | 找Bug并修复 | 排查为什么登录失败 |
| **Refactor** | 重构 | 改善代码结构但不改变功能 | 整理混乱的代码 |
| **Deploy** | 部署 | 把代码发布到线上 | 上线新版本 |
| **Build** | 构建 | 把代码打包成可运行的程序 | npm build |
| **Lint** | 代码检查 | 检查代码是否符合规范 | ESLint检查语法错误 |
| **Runtime** | 运行时 | 代码执行的环境 | Node.js运行环境 |
| **Dependency** | 依赖 | 项目需要的第三方库 | React、Lodash |
| **Framework** | 框架 | 提供基础结构的开发工具 | React、Vue、Spring Boot |
| **Library** | 库 | 提供特定功能的代码集合 | jQuery、Moment.js |

## 三、前端开发类

| 名词 | 含义 | 通俗解释 | 使用场景 |
|------|------|---------|---------|
| **DOM** | 文档对象模型 | 浏览器理解的页面结构 | 修改页面元素内容 |
| **Component** | 组件 | 可复用的界面模块 | 一个按钮、一个卡片 |
| **State** | 状态 | 数据在某个时刻的值 | 用户是否已登录 |
| **Props** | 属性 | 父组件传给子组件的数据 | 传给按钮的文字 |
| **Hook** | 钩子 | 在React中使用状态等功能的函数 | useState、useEffect |
| **Route** | 路由 | 页面地址和内容的对应关系 | /home 显示首页 |
| **CSS** | 层叠样式表 | 控制页面外观的代码 | 设置颜色、字体、布局 |
| **Responsive** | 响应式 | 自动适应不同屏幕大小 | 手机和电脑显示不同布局 |
| **API Call** | API调用 | 前端向后端请求数据 | 获取用户信息 |
| **Event Handler** | 事件处理器 | 处理用户操作的代码 | 点击按钮后做什么 |
| **Virtual DOM** | 虚拟DOM | 内存中的页面副本，提升性能 | React的优化机制 |

## 四、后端开发类

| 名词 | 含义 | 通俗解释 | 使用场景 |
|------|------|---------|---------|
| **Server** | 服务器 | 运行后端程序的计算机 | 处理用户请求 |
| **Database** | 数据库 | 存储数据的系统 | 存用户信息、订单 |
| **Query** | 查询 | 从数据库获取数据 | SELECT * FROM users |
| **Middleware** | 中间件 | 请求和响应之间的处理层 | 验证用户是否登录 |
| **Endpoint** | 端点 | API的具体地址 | /api/users |
| **Request** | 请求 | 客户端发给服务器的消息 | GET /api/users |
| **Response** | 响应 | 服务器返回给客户端的消息 | { "name": "张三" } |
| **Authentication** | 认证 | 验证你是谁 | 登录验证 |
| **Authorization** | 授权 | 验证你能做什么 | 管理员才能删除 |
| **CORS** | 跨域资源共享 | 允许不同域名之间的请求 | 前端调用后端API |
| **Session** | 会话 | 用户和服务器的一次交互周期 | 登录后保持登录状态 |
| **Cache** | 缓存 | 临时存储，加速访问 | 把热门数据放内存 |

## 五、AI/大模型类

| 名词 | 含义 | 通俗解释 | 使用场景 |
|------|------|---------|---------|
| **LLM** | 大语言模型 | 能理解和生成文本的AI | ChatGPT、Claude |
| **Token** | 词元 | AI处理文本的最小单位 | 1个Token约0.75个英文单词 |
| **Context Window** | 上下文窗口 | AI能"记住"的最大文本量 | GPT-4约128K Token |
| **Fine-tuning** | 微调 | 用特定数据训练模型 | 让模型学会特定领域知识 |
| **RAG** | 检索增强生成 | 先搜索再回答 | 企业知识库问答 |
| **Embedding** | 嵌入向量 | 把文字变成数字，表示含义 | 语义搜索、相似度计算 |
| **Vector DB** | 向量数据库 | 存储嵌入向量的数据库 | Milvus、Pinecone |
| **Agent** | 智能体 | 能自主完成任务的AI | 自动写代码+测试+修复 |
| **Prompt Engineering** | 提示词工程 | 设计好的Prompt让AI输出更好 | 角色设定+任务描述+示例 |
| **Chain of Thought** | 思维链 | 让AI逐步推理 | "请一步步思考..." |
| **Few-shot** | 少样本学习 | 给AI几个例子让它学 | 示例1、示例2、请回答 |
| **Hallucination** | 幻觉 | AI编造不存在的信息 | AI说了一个错误的事实 |
| **Temperature** | 温度 | 控制输出的随机性 | 0=精确，1=创意 |
| **Function Calling** | 函数调用 | 让AI调用外部工具 | AI调用天气API |
| **Streaming** | 流式输出 | 逐字生成回复 | ChatGPT一个字一个字蹦 |
| **System Prompt** | 系统提示词 | 给AI设定角色和规则 | "你是一个专业的翻译" |
| **Grounding** | 落地/ grounding | 让AI的回答基于事实 | 基于搜索结果回答 |

## 八、常见AI编程指令模板

### 1. 代码生成类

```
# 描述需求
"帮我写一个[功能]，使用[技术栈]，要求[具体要求]"

# 示例
"帮我写一个用户登录页面，使用React+TailwindCSS，要求包含邮箱和密码输入框、表单验证、提交按钮"
```

### 2. 代码解释类

```
# 理解陌生代码
"解释这段代码的作用：[粘贴代码]"

# 示例
"解释这个useEffect钩子的作用，它依赖了哪些变量？"
```

### 3. Bug修复类

```
# 描述问题
"我遇到了[错误信息]，代码是：[粘贴代码]，请帮我修复"

# 示例
"我遇到了TypeError: Cannot read property 'name' of undefined，代码是..."
```

### 4. 代码优化类

```
# 提升质量
"优化这段代码，使其[更快/更简洁/更易读/更安全]"

# 示例
"优化这段代码，使其性能更好，减少不必要的重复计算"
```

### 5. 测试编写类

```
# 生成测试
"为这段代码编写单元测试，覆盖[正常情况/边界情况/异常情况]"

# 示例
"为这个登录函数编写单元测试，覆盖密码正确、密码错误、网络异常等情况"
```

## 十、快速问答

**Q：AI说"Token不够了"是什么意思？**  
A：AI的单次对话有字数上限，需要开启新对话或精简内容。

**Q：AI生成的代码有Bug怎么办？**  
A：把错误信息告诉AI，让它修复。AI编程助手不是100%准确，需要人工审查。

**Q：@文件是什么意思？**  
A：在AI编程工具中，@文件名可以让AI读取并理解那个文件的内容。

**Q：什么是"上下文"？**  
A：AI能看到的你的项目信息。上下文越多，AI的理解越准确，但也会消耗更多Token。

**Q：AI编程会取代程序员吗？**  
A：AI是辅助工具，能大幅提升效率，但复杂架构设计、业务理解、调试等仍需人类。
