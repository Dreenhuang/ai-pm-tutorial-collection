## 📖 核心内容

### 一、为什么需求阶段最需要AI？

**通俗比喻**：做产品需求就像做菜前准备食材。如果食材不新鲜、搭配不合理，后面厨艺再好也做不出好菜。需求阶段就是"选食材、定菜谱"的环节，而AI就是你的"智能厨房助手"。

在传统需求工作中，产品经理常面临三大痛点：

| 痛点 | 传统做法 | AI辅助后 |
|------|---------|---------|
| 信息过载 | 手动整理上百条用户反馈 | AI自动聚类、提取高频需求 |
| 需求遗漏 | 依赖个人经验判断 | AI基于历史数据提示常见盲区 |
| 优先级争议 | 主观拍脑袋决定 | AI提供数据支撑的排序建议 |

### 二、AI辅助需求收集

#### 1. 用户反馈智能分析

**工作流**：

```
原始用户反馈 → AI文本清洗 → 情感分析 → 需求聚类 → 优先级排序
```

**实操示例**：

将收集到的用户反馈输入AI（如ChatGPT、文心一言），使用以下提示词：

```
你是一位资深产品经理。请帮我分析以下用户反馈：
1. 提取所有功能需求点
2. 按情感倾向分类（正面/负面/中性）
3. 将相似需求合并去重
4. 按出现频率排序
5. 输出结构化表格

用户反馈列表：
[粘贴你的用户反馈]
```

**输出示例**：

| 需求点 | 出现次数 | 情感倾向 | 用户原话摘要 |
|--------|---------|---------|-------------|
| 夜间模式 | 23次 | 负面（缺失抱怨） | "晚上看太刺眼" |
| 数据导出 | 18次 | 中性 | "希望能导出Excel" |
| 搜索优化 | 15次 | 负面 | "搜不到想要的" |

#### 2. 竞品需求矩阵生成

**提示词模板**：

```
请对比以下5个竞品的功能差异，输出功能矩阵表格：
产品A：[描述]
产品B：[描述]
...

要求：
- 列出核心功能模块
- 标注每个产品是否有该功能
- 分析功能差异背后的用户价值
- 找出市场空白点
```

### 三、AI辅助需求分析

#### 1. 用户画像智能生成

传统做法：手动调研、访谈、整理，耗时2-3周。  
AI做法：输入基础数据，AI在30分钟内生成初步画像。

**提示词模板**：

```
基于以下数据，生成3个典型用户画像：
- 产品：[产品名称]
- 目标用户：[描述]
- 已知特征：[年龄/职业/使用场景等]

输出格式：
1. 姓名、年龄、职业
2. 核心需求（按优先级）
3. 使用场景（早中晚/工作日周末）
4. 痛点与期望
5. 决策影响因素
```

#### 2. 需求优先级AI辅助评估

**Kano模型AI辅助**：

```
请用Kano模型分析以下需求，分类为：
- 基本型需求（必须有）
- 期望型需求（越多越好）
- 兴奋型需求（超出预期）
- 无差异需求（可有可无）
- 反向需求（做了反而不好）

需求列表：[列出需求]

输出表格并说明分类理由。
```

### 四、AI辅助PRD文档生成

**标准PRD框架提示词**：

```
请帮我生成一份标准PRD文档框架，包含以下章节：
1. 文档信息（版本历史、审批记录）
2. 产品概述（背景、目标、范围）
3. 用户画像与场景
4. 功能需求清单（含优先级）
5. 非功能需求（性能、安全、兼容）
6. 数据埋点需求
7. 上线计划与验收标准
8. 风险与依赖项

产品背景：[简述你的产品]
目标用户：[描述]
核心功能：[列出3-5个核心功能]

要求：
- 每个章节给出具体的填写指引
- 提供示例内容
- 标注需要产品经理重点确认的部分
```

### 五、AI辅助需求的常见陷阱

| 陷阱 | 表现 | 避免方法 |
|------|------|---------|
| AI幻觉 | 编造不存在的需求或数据 | 所有AI输出必须人工验证 |
| 过度依赖 | 直接用AI输出不做二次加工 | AI只是助手，决策权在你 |
| 语境缺失 | AI不理解公司内部术语 | 提供充分的背景信息和术语表 |
| 时效性差 | AI知识库有截止日期 | 最新市场数据需自行补充 |

### 六、实操工作流：从零到PRD

```
第1步：AI收集用户需求关键词（30分钟）
   ↓
第2步：AI生成用户画像和场景（30分钟）
   ↓
第3步：AI辅助需求优先级排序（30分钟）
   ↓
第4步：AI生成PRD框架和内容初稿（1小时）
   ↓
第5步：人工审核、补充、调整（2小时）
   ↓
第6步：团队评审定稿（1小时）
```

**时间对比**：传统方式约15-20小时，AI辅助后约5-6小时，效率提升约70%。

## 📦 案例分析

### 案例1：某电商APP的搜索功能优化

**背景**：用户反馈"搜索找不到想要的商品"，投诉率月增15%。

**AI辅助过程**：
1. 将1万条用户反馈输入AI进行聚类分析
2. AI识别出3大核心问题：同义词不识别、错别字不纠正、筛选条件不直观
3. 用Kano模型分析，确定"同义词识别"为基本需求，"搜索历史推荐"为兴奋需求
4. AI生成PRD初稿，包含功能描述、交互流程、验收标准

**结果**：需求评审时间从3天缩短到1天，开发团队理解成本降低50%。

### 案例2：某SaaS产品的需求优先级争议

**背景**：销售团队、客服团队、技术团队对需求优先级意见不一，僵持2周。

**AI辅助过程**：
1. 收集各方需求，输入AI进行RICE评分（Reach影响范围、Impact影响程度、Confidence信心、Effort投入成本）
2. AI输出客观排序，发现"数据导出"功能的RICE分数最高
3. 基于AI数据，各方达成共识

**结果**：1天内解决2周争议，需求排期效率提升10倍。

## 💡 产品经理视角的关键思考

1. **AI不会替代需求洞察**：AI能处理数据，但"用户为什么需要这个"的洞察只能来自你对用户的理解。
2. **质量输入=质量输出**：给AI的反馈数据质量决定了AI分析的质量。垃圾进，垃圾出。
3. **建立AI辅助标准流程**：不要每次临时想提示词，建立团队的AI辅助需求分析模板库。
4. **保持批判性思维**：AI给出的优先级排序只是参考，最终决策需要结合业务战略和资源情况。

3. 当AI的优先级排序与你的直觉判断不一致时，你会如何处理？
