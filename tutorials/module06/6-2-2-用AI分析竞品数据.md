## 📖 核心内容

### 一、竞品分析的产品经理视角

**通俗比喻**：竞品分析就像"逛菜市场"。你要做的不是抄别人卖什么，而是搞清楚：别人为什么卖得好？顾客为什么愿意在某个摊位前停留？什么菜在这个季节最受欢迎？

竞品分析的核心目的：

| 目的 | 说明 | 产品经理的关注点 |
|------|------|----------------|
| 了解市场格局 | 谁在玩？怎么玩？ | 市场机会在哪里 |
| 发现用户需求 | 用户在竞品中得到了什么/没得到什么 | 我们的差异化价值 |
| 验证产品方向 | 我们的方向是否靠谱 | 竞品验证了什么，避开了什么 |
| 学习优秀实践 | 别人做对了什么 | 可借鉴的交互、功能、策略 |

### 二、竞品数据采集的AI辅助方法

#### 1. 公开信息智能收集

**提示词模板**：

```
请帮我收集以下竞品的公开信息：
竞品名称：[列出3-5个竞品]
关注维度：
- 产品定位和目标用户
- 核心功能和差异化特点
- 商业模式和定价策略
- 用户规模和融资历程
- 最近3个月的重大更新

请整理为对比表格，标注信息来源。
```

#### 2. 用户评价智能分析

**提示词模板**：

```
请分析以下竞品的用户评价：
竞品：[名称]
数据来源：[应用商店评论、社交媒体、论坛等]
评价样本：[粘贴评价数据]

分析要求：
1. 用户最满意的3个方面
2. 用户最不满意的3个方面
3. 用户提到但未满足的需求
4. 评价的情感趋势（近期变化）
5. 竞品间用户评价的对比

输出格式：分析报告+对比表格
```

#### 3. 功能对比矩阵生成

**提示词模板**：

```
请生成以下竞品的功能对比矩阵：
竞品：[列出]
产品类别：[描述]

对比维度：
1. 核心功能（按模块列出）
2. 特色功能
3. 用户体验（注册流程、核心路径步骤数）
4. 技术特点（支持平台、更新频率）
5. 商业化功能（付费点、定价模式）

输出格式：
- 功能矩阵表格（行=功能，列=竞品，标记是否有）
- 功能差异总结
- 市场空白点识别
```

**输出示例**：

| 功能模块 | 竞品A | 竞品B | 竞品C | 市场空白 |
|---------|-------|-------|-------|---------|
| 用户注册 | 手机号/邮箱 | 微信一键登录 | 手机号 | 社交账号绑定 |
| 核心功能1 | ✅ | ✅ | ✅ | - |
| 核心功能2 | ✅ | ❌ | ✅ | 竞品B缺失 |
| 特色功能 | AI推荐 | 社交分享 | 数据导出 | 跨平台同步 |
| 付费模式 | 订阅制 | 买断制 | 免费+内购 | 企业定制 |

### 三、AI辅助竞品深度分析

#### 1. 竞品战略意图分析

**提示词模板**：

```
请分析以下竞品的战略意图：
竞品：[名称]
近期动态：[融资、产品更新、市场动作等]
业务布局：[描述]

分析要求：
1. 该竞品试图占领的市场位置
2. 其核心竞争策略（价格战？差异化？生态？）
3. 其优势壁垒和劣势短板
4. 对我们产品的潜在威胁等级
5. 建议的应对策略

输出格式：战略分析简报
```

#### 2. 竞品用户群体分析

**提示词模板**：

```
请分析以下竞品的目标用户群体：
竞品：[名称]
已知信息：[用户画像、使用场景等]

分析要求：
1. 核心用户画像（年龄、职业、需求、痛点）
2. 用户获取渠道
3. 用户留存策略
4. 用户付费转化路径
5. 与我们的产品用户群体的重合度和差异点

输出格式：用户群体分析报告
```

#### 3. 竞品迭代趋势分析

**提示词模板**：

```
请分析以下竞品的产品迭代趋势：
竞品：[名称]
版本历史：[列出近1年的主要版本更新]

分析要求：
1. 迭代频率和节奏
2. 每次迭代的核心方向（功能增强？体验优化？商业化？）
3. 迭代方向背后的战略意图
4. 未来可能的迭代方向预测
5. 对我们的启示

输出格式：迭代趋势分析报告
```

### 四、竞品分析框架

#### 1. 常用分析框架对比

| 框架 | 适用场景 | 核心维度 | AI辅助点 |
|------|---------|---------|---------|
| SWOT分析 | 快速了解竞品优劣势 | 优势、劣势、机会、威胁 | AI生成完整SWOT矩阵 |
| 波特五力 | 行业竞争格局分析 | 供应商、买家、替代品、新进入者、现有竞争者 | AI填充各维度信息 |
| 用户体验地图 | 用户视角竞品分析 | 触点、行为、想法、情绪 | AI生成地图初稿 |
| 商业模式画布 | 竞品商业模式分析 | 价值主张、客户细分、渠道、收入等 | AI生成画布内容 |
| 功能对比矩阵 | 功能差异分析 | 功能有无、实现质量 | AI自动抓取和对比 |

#### 2. AI辅助SWOT分析

**提示词模板**：

```
请为以下竞品生成SWOT分析：
竞品：[名称]
产品信息：[描述]
市场环境：[描述]

要求：
- 每个维度列出3-5个具体点
- 每个点有具体依据（数据、用户反馈等）
- 分析对我们的启示和应对建议

输出格式：
S（优势）：
- [具体点] + 依据
W（劣势）：
- [具体点] + 依据
O（机会）：
- [具体点] + 依据
T（威胁）：
- [具体点] + 依据
```

### 五、竞品数据可视化

#### 1. 竞品雷达图

**提示词模板**：

```
请为以下竞品生成雷达图评估：
竞品：[列出3-5个]
评估维度（5-7个）：
- 功能完整性
- 用户体验
- 品牌影响力
- 市场份额
- 技术创新
- 用户口碑
- 商业化能力

请为每个维度打分（1-10分），并说明打分依据。
输出各竞品的雷达图数据。
```

#### 2. 竞品市场定位图

**提示词模板**：

```
请生成竞品的市场定位分析：
竞品：[列出]
X轴维度：[如产品价格（低→高）]
Y轴维度：[如功能复杂度（简单→复杂）]

请：
1. 将各竞品放入坐标系
2. 分析各竞品占据的市场位置
3. 识别市场空白区域
4. 建议我们产品的定位

输出格式：文字描述+坐标表格
```

### 六、从竞品分析到产品策略

#### 策略转化工作流

```
竞品数据收集 → AI分析整理 → 发现市场机会 → 制定产品策略 → 验证策略可行性
```

**提示词模板**：

```
请基于以下竞品分析结果，为我产品制定策略建议：
竞品分析摘要：[总结]
我们产品的优势：[列出]
我们产品的劣势：[列出]
可用资源：[描述]

请输出：
1. 差异化定位建议
2. 核心功能优先级建议
3. 避免直接竞争的策略
4. 可以借鉴的竞品实践
5. 短期（3个月）和长期（1年）策略

输出格式：产品策略建议报告
```

### 七、竞品分析的常见陷阱

| 陷阱 | 表现 | 避免方法 |
|------|------|---------|
| 照抄竞品 | 竞品有什么我们就做什么 | 分析"为什么"再做，而非"有什么"就抄 |
| 静态分析 | 只看当前状态，不看趋势 | 关注竞品迭代历史和方向 |
| 忽略用户视角 | 只比功能，不比体验 | 加入用户体验维度分析 |
| 过度解读 | 从少量信息推断过多 | 用多源数据交叉验证 |
| 忽视小竞品 | 只盯头部竞品 | 关注新兴竞品和垂直竞品 |

## 📦 案例分析

### 案例1：某笔记APP的竞品分析

**背景**：新产品计划进入笔记市场，需分析Notion、印象笔记、语雀等竞品。

**AI辅助过程**：
1. AI收集5个主流笔记产品的功能矩阵，发现"AI辅助写作"是市场空白
2. AI分析用户评论，发现用户对"跨平台同步稳定性"抱怨最多
3. AI生成SWOT分析，识别出小团队的机会在于"垂直场景+AI增强"
4. 基于分析结果，确定产品定位为"AI增强的垂直场景笔记"

**结果**：产品上线后，凭借AI写作辅助功能获得差异化竞争优势。

### 案例2：某跨境电商工具的竞品追踪

**背景**：产品运营1年后，需要持续追踪竞品动态。

**AI辅助过程**：
1. 建立竞品追踪机制：每月用AI抓取竞品更新信息
2. AI分析竞品版本更新日志，识别趋势变化
3. 发现某竞品开始加大"数据分析"功能投入
4. AI评估威胁等级，建议加快自身数据分析功能的开发

**结果**：提前3个月启动数据分析功能开发，保持竞争优势。

## 💡 产品经理视角的关键思考

1. **竞品分析不是"抄作业"，而是"找答案"**：竞品做了什么不重要，重要的是它为什么这么做、用户是否买账。
2. **关注用户，而非竞品**：最终决定产品成败的是用户，不是竞品。竞品分析的目的是更好理解用户。
3. **定期更新，而非一次性工作**：市场在变，竞品在变，竞品分析应该是持续的过程。
4. **行动比报告重要**：再漂亮的竞品分析报告，如果不转化为产品行动，就只是纸上谈兵。
