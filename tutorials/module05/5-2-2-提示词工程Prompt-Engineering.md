## 📖 核心内容

### 一、什么是提示词（Prompt）？

**定义**：输入给大语言模型的文本指令，用于引导模型生成期望的输出。

**通俗比喻**：Prompt就像"给员工的任务说明书"。说明书写得越清楚，员工完成得越好；模糊的指令只会得到模糊的结果。

### 二、Prompt的核心结构

一个高质量的Prompt通常包含以下要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **角色设定** | 定义AI的身份和专业领域 | "你是一位资深的前端工程师" |
| **任务描述** | 明确要做什么 | "请为以下用户故事编写验收标准" |
| **上下文信息** | 提供必要的背景知识 | "我们的产品是一个电商平台，目标用户是..." |
| **输出格式** | 规定输出的结构和格式 | "请以表格形式输出，包含以下列..." |
| **约束条件** | 设定边界和限制 | "不要使用专业术语，确保初中文化水平能看懂" |
| **示例（Few-shot）** | 提供示范帮助理解 | "例如：输入A，输出B" |

### 三、好的Prompt vs 差的Prompt

| 差的Prompt | 好的Prompt |
|-----------|-----------|
| "帮我写个产品文档" | "你是一位有10年经验的SaaS产品经理。请为以下功能撰写PRD：[功能描述]。输出包含：背景、用户故事、功能需求、非功能需求、验收标准。每个部分用二级标题分隔，使用简洁专业的语言。" |
| "分析这个数据" | "你是一位数据分析师。请分析以下销售数据，找出：1）环比增长最高的产品类别；2）可能的增长原因；3）下个月的预测。数据：[数据]。输出用表格+要点形式。" |
| "总结这篇文章" | "请用3句话总结以下文章的核心观点，每句话不超过20字。文章：[内容]" |

### 四、常用Prompt设计框架

#### 1. CREATE框架

| 字母 | 含义 | 说明 |
|------|------|------|
| **C** | Character（角色） | 设定AI的身份 |
| **R** | Request（请求） | 明确任务 |
| **E** | Examples（示例） | 提供示范 |
| **A** | Adjustments（调整） | 设定约束 |
| **T** | Type（类型） | 规定输出格式 |
| **E** | Extras（补充） | 额外要求 |

#### 2. CRISPE框架

| 字母 | 含义 | 说明 |
|------|------|------|
| **C** | Capacity（能力） | AI的角色和专业知识 |
| **R** | Role（角色） | 具体身份设定 |
| **I** | Insight（洞察） | 背景信息和上下文 |
| **S** | Statement（陈述） | 具体任务描述 |
| **P** | Personality（个性） | 输出风格和语气 |
| **E** | Experiment（实验） | 输出格式和约束 |

#### 3. 产品经理常用模板

```
# 角色
你是一位经验丰富的[角色]，擅长[专业技能]。

# 背景
[产品/项目背景信息]

# 任务
请完成以下任务：[具体任务描述]

# 要求
1. [要求1]
2. [要求2]
3. [要求3]

# 输出格式
请以[格式]输出，包含以下部分：
- [部分1]
- [部分2]
- [部分3]

# 约束
- [约束1]
- [约束2]
```

### 五、高级Prompt技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| **Zero-shot（零样本）** | 不给示例，直接给指令 | "请总结以下文章" |
| **Few-shot（少样本）** | 给2-3个示例 | "例如：输入A→输出B；输入C→输出D。现在请处理：输入E" |
| **Chain of Thought（思维链）** | 让模型"逐步思考" | "请逐步分析这个问题，展示你的推理过程" |
| **Self-Consistency（自洽性）** | 多次生成取最一致结果 | 同一问题问3次，选出现最多的答案 |
| **Role-Play（角色扮演）** | 让模型扮演特定角色 | "假设你是一位对价格敏感的用户..." |

### 六、Prompt在产品中的应用

| 产品场景 | Prompt策略 | 关键考量 |
|---------|-----------|---------|
| **智能客服** | 固定模板+动态变量 | 一致性、安全性、品牌语气 |
| **内容生成** | 灵活模板+多样化参数 | 创意性、多样性、避免重复 |
| **代码助手** | 技术约束+格式要求 | 准确性、安全性、可维护性 |
| **数据分析** | 结构化指令+数据格式 | 准确性、可解释性 |

### 七、Prompt管理的最佳实践

| 实践 | 说明 | 工具建议 |
|------|------|---------|
| **版本控制** | Prompt像代码一样需要版本管理 | Git、专用Prompt管理工具 |
| **A/B测试** | 对比不同Prompt的效果 | 记录每次实验的结果 |
| **模板库** | 积累可复用的Prompt模板 | 建立团队共享的Prompt库 |
| **效果追踪** | 监控Prompt输出的质量 | 建立评估指标体系 |
| **持续优化** | 根据反馈迭代改进 | 定期回顾和更新 |

## 📦 案例分析

### 案例1：Notion AI的Prompt设计

**背景**：Notion AI提供多种写作辅助功能，每种功能背后都有精心设计的Prompt。

| 功能 | Prompt设计要点 | 效果 |
|------|---------------|------|
| **续写** | 基于当前上下文，保持语气一致 | 自然流畅的续写 |
| **总结** | "用要点形式总结，不超过5条" | 简洁清晰的摘要 |
| **改写** | "保持原意，改为更[正式/随意]的语气" | 风格可控的改写 |
| **翻译** | "翻译为[语言]，保持专业术语准确" | 高质量的翻译 |

**产品经理启示**：一个AI功能的核心竞争力往往在于Prompt的设计质量，而非模型本身。

### 案例2：某公司的Prompt管理实践

**背景**：某电商公司有50+个AI功能，每个功能使用不同的Prompt。

**挑战**：
- Prompt散落在代码各处，难以维护
- 不同团队使用相似但不同的Prompt，效果不一致
- 无法追踪哪个Prompt效果更好

**解决方案**：
1. 建立中央Prompt库，所有Prompt集中管理
2. 每个Prompt关联使用场景、版本、效果数据
3. 建立Prompt评审流程，上线前必须经过测试
4. 定期回顾低效Prompt，持续优化

**效果**：Prompt维护效率提升60%，AI功能用户满意度提升25%。

## 💡 产品经理视角的关键思考

1. **Prompt是产品的核心资产**：就像配方之于餐厅，Prompt的质量直接决定AI产品的体验。
2. **没有"万能Prompt"**：不同场景、不同模型、不同需求需要不同的Prompt设计。
3. **Prompt需要持续迭代**：用户反馈、模型更新、业务变化都需要Prompt随之调整。
4. **团队需要Prompt工程能力**：这不是"随便写几句话"，而是需要系统化学习和实践的技能。
