## 📖 核心内容

### 一、什么是大语言模型（LLM）？

**定义**：Large Language Model，基于海量文本数据训练的超大规模神经网络模型，能够理解、生成和处理自然语言。

**通俗比喻**：LLM就像一个"读过几乎所有书、看过几乎所有文章"的超级学霸。它不是"知道"答案，而是根据学习到的语言规律，预测下一个最可能出现的词。

### 二、LLM的核心工作原理

#### 1. Tokenization（分词）

**概念**：将输入文本切分成模型能理解的最小单元（Token）。

| 输入文本 | Token切分示例 | Token数量 |
|---------|-------------|----------|
| "你好世界" | ["你", "好", "世", "界"] | 4 |
| "Hello World" | ["Hello", " World"] | 2 |
| "人工智能改变未来" | ["人工", "智能", "改变", "未来"] | 4 |

**产品经理须知**：
- 中文通常1个汉字≈1个Token
- 英文1个单词≈1-2个Token
- Token数量直接影响调用成本和处理速度

#### 2. 预测下一个Token

**核心机制**：LLM本质是一个"下一个词预测器"。

```
输入: "今天天气真" 
模型计算每个候选词的概率:
- "好": 85%
- "不": 8%
- "热": 5%
- "冷": 2%
输出: "好" (或根据采样策略选择)

新的输入: "今天天气真好" → 继续预测下一个词...
```

**通俗比喻**：就像你玩"成语接龙"，但不是接最后一个字，而是根据前面的所有内容，猜下一个最合理的词。

#### 3. Transformer架构（简化理解）

| 核心组件 | 功能 | 通俗比喻 |
|---------|------|---------|
| **Self-Attention（自注意力）** | 让模型理解词与词之间的关系 | 读文章时，大脑自动关联前后文的关键信息 |
| **Positional Encoding（位置编码）** | 让模型知道词的顺序 | 记住"我打他"和"他打我"意思完全不同 |
| **Feed-Forward Network（前馈网络）** | 对信息进行处理和转化 | 大脑对信息进行思考和加工 |
| **Layer Normalization（层归一化）** | 稳定训练过程 | 保持信息处理的一致性 |

#### 4. 上下文窗口（Context Window）

**定义**：模型一次性能够处理的输入+输出的最大Token数量。

| 模型 | 上下文窗口 | 约等于处理能力 |
|------|-----------|---------------|
| GPT-3.5 | 4K-16K tokens | 约3千-1.2万字 |
| GPT-4 | 8K-128K tokens | 约6千-9.6万字 |
| Claude 3 | 200K tokens | 约15万字 |
| 文心一言4.0 | 128K tokens | 约9.6万字 |

**产品经理须知**：
- 上下文窗口≠记忆能力，是"一次性工作记忆"
- 长文档处理需要考虑窗口限制
- 超出窗口的信息模型会"遗忘"

### 三、关键参数详解

#### 1. Temperature（温度）

控制输出的随机性和创造性。

| Temperature值 | 效果 | 适用场景 |
|--------------|------|---------|
| 0 | 确定性最高，每次输出相同 | 代码生成、事实性问答 |
| 0.2-0.4 | 偏保守，输出稳定 | 客服回答、产品说明 |
| 0.7-0.8 | 平衡创造性和准确性 | 创意写作、头脑风暴 |
| 1.0+ | 高度创造性，可能偏离主题 | 诗歌创作、趣味对话 |

**通俗比喻**：Temperature就像"创意旋钮"，调得越低越保守，调得越高越放飞。

#### 2. Top-P（核采样）

控制模型从多少个候选词中选择。

| Top-P值 | 含义 | 效果 |
|---------|------|------|
| 0.1 | 只从概率最高的10%候选词中选择 | 输出保守、安全 |
| 0.5 | 从50%候选词中选择 | 平衡 |
| 0.9 | 从90%候选词中选择 | 更多样化 |
| 1.0 | 从所有候选词中选择 | 最大随机性 |

#### 3. Max Tokens（最大输出长度）

限制模型单次生成的最大Token数量。

**注意**：设置过小可能导致回答被截断；设置过大可能浪费成本。

## 📦 案例分析

### 案例1：ChatGPT的对话能力来源

**问题**：为什么ChatGPT能进行自然的多轮对话？

**原理解析**：
1. **上下文拼接**：每轮对话，系统会将历史对话+当前问题一起发送给模型
2. **角色提示**：通过System Prompt设定助手的行为模式
3. **温度调优**：日常对话通常使用0.7-0.8的Temperature，保持自然但有边界

```
实际发送给模型的完整内容：
[System] 你是一个有帮助的助手。
[User] 你好，我想学Python
[Assistant] 你好！学习Python是个很好的选择...
[User] 那我应该从哪里开始？
→ 模型看到完整历史，生成连贯回复
```

**产品启示**：多轮对话的连贯性不是模型"记住"了什么，而是系统把历史记录重新发送给了模型。

### 案例2：Notion AI的辅助写作

**功能分析**：Notion AI利用LLM实现以下能力：

| 功能 | 底层原理 | 关键Prompt设计 |
|------|---------|---------------|
| 续写 | 基于上文预测下文 | "请根据以下内容继续撰写..." |
| 总结 | 信息压缩提取 | "请用3句话总结以下内容的要点..." |
| 翻译 | 跨语言映射 | "请将以下中文翻译为英文，保持专业术语准确..." |
| 改写 | 风格迁移 | "请将以下内容改写为更正式的商业语气..." |

**产品启示**：同一个模型，通过不同的Prompt设计，可以实现多种产品功能。Prompt工程是AI产品的核心能力。

## 💡 产品经理视角的关键思考

1. **LLM不是搜索引擎**：它不"知道"事实，而是基于训练数据生成最可能的回答。这意味着它可能生成看似合理但实际错误的内容（幻觉）。
2. **参数调优是产品体验的关键**：同样的模型，不同的Temperature和Prompt，体验天差地别。
3. **成本模型要提前算清**：每次调用都按Token计费，高频场景的成本可能超出预期。
4. **上下文窗口限制产品设计**：长文档处理需要额外的技术方案（如分块、摘要、RAG）。
